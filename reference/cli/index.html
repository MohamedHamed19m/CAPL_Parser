    CLI Reference

    The capl_tools command-line interface allows you to scan and modify CAPL files quickly.

    capl_tools

    A powerful CLI for parsing and manipulating CAPL files.

    Usage

    capl_tools [OPTIONS] COMMAND [ARGS]...

    Arguments

    No arguments available

    Options

    Name Description Required Default
    --help Show this message and exit. No -

    Commands

    Name Description
    scan List the logical structure of a CAPL file...
    stats Get a highly compressed inventory of the...
    remove Remove a specific element by its type and...
    get Fetch the raw code of a specific element.
    insert Insert code at specified location using...

    Sub Commands

    capl_tools scan

    List the logical structure of a CAPL file (Names, Signatures, and Types).

    Usage

    capl_tools scan [OPTIONS] PATH

    Arguments

    Name Description Required
    PATH Path to the .can file Yes

    Options

    Name Description Required Default
    --toon Output structure in TOON No -
    --full Include physical line numbers in the output No -
    --help Show this message and exit. No -

    capl_tools stats

    Get a highly compressed inventory of the CAPL file content.

    Usage

    capl_tools stats [OPTIONS] PATH

    Arguments

    Name Description Required
    PATH Path to the .can file Yes

    Options

    Name Description Required Default
    -m, --machine Single-line output (e.g., TestCase:15 Function:3) No
    --help Show this message and exit. No -

    capl_tools remove

    Remove a specific element by its type and name.

    Usage

    capl_tools remove [OPTIONS] PATH

    Arguments

    Name Description Required
    PATH Path to the .can file Yes

    Options

    Name Description Required Default
    -t, --type [TestCase|Function|Handler|TestFunction|CaplInclude|CaplVariable|TestGroup] Type of element to remove Yes -
    -n, --name TEXT Name of the element to remove Yes -
    --help Show this message and exit. No -

    capl_tools get

    Fetch the raw code of a specific element.

    Usage

    capl_tools get [OPTIONS] PATH NAME

    Arguments

    Name Description Required
    PATH Path to the .can file Yes
    NAME Name of the element to fetch Yes

    Options

    Name Description Required Default
    -t, --type TEXT Type of element (e.g. TestCase, Function, Handler) Yes -
    --help Show this message and exit. No -

    capl_tools insert

    Insert code at specified location using semantic anchoring.

    Usage

    capl_tools insert [OPTIONS] PATH

    Arguments

    Name Description Required
    PATH Path to the .can file Yes

    Options

    Name Description Required Default
    -l, --location TEXT Location: 'after:', 'section:', or 'line:' Yes -
    -t, --type [TestCase|Function|Handler|TestFunction|CaplInclude|CaplVariable|TestGroup] Type of element being inserted No -
    -s, --source PATH Path to code snippet (omit to read from stdin) No -
    -c, --code TEXT Code string to insert directly No -
    --help Show this message and exit. No -
